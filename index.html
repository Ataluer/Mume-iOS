使用说明

代理

代理是指用户的手机网络请求可以通过一个中间服务到达远程服务器，在某些情况下以此实现穿越局域网防火墙的作用。

Potatso 支持自定义代理，需要由用户自行提供（自己架设或购买成品服务）。

目前支持的代理类型包括：

  • Shadowsocks
  • Shadowsocksr
  • SOCKS5

规则

Potatso 支持自定义规则，通过一系列规则，定制自己的网络环境。通过规则，用户可以对指定的网络请求执行特定的操作（直连、拒绝、代理），以此达到屏蔽广告、穿越防火墙等功能。

规则支持匹配 DOMAIN、DOMAIN-SUFFIX、DOMAIN-MATCH，IP-CIDR，GEOIP 等关键词：

DOMAIN：完全匹配请求域名

如 DOMAIN, some.example.com 会匹配 some.example.com, 但不会匹配 example.com, www.example.com, hsome.example.com

DOMAIN-SUFFIX：匹配请求的域名后缀，一般用于匹配子域名

如 DOMAIN-SUFFIX, example.com 会匹配 example.com, some.example.com, www.test.example.com, 但不会匹配 example.some.com, bexample.com

DOMAIN-MATCH：匹配满足给定关键词的域名

如 DOMAIN-SUFFIX, google 会匹配 www.google.com, google.com.hk, googleusercontent.com, 但不会匹配 goo.gle.com

IP-CIDR：匹配处在给定域内的 IP

如 IP-CIDR, 17.0.0.0/8 会匹配 17.0.0.1, 17.17.17.17, 但不会匹配 18.0.0.0

GEOIP：匹配给定地理区域内的 IP (使用 Country Code 指定)

如 GEOIP, CN 会匹配中国地区的 IP

支持以下动作：

  • DIRECT: 直接访问
  • PROXY: 通过代理访问
  • REJECT: 拒绝

规则集

规则集是一系列相似规则的集合，如广告屏蔽规则集、无图模式规则集、中国地区直连规则集，设立规则集的目的是为了方便用户更加灵活地最小化组织、分享规则。

配置组

配置组是代理和规则集的搭配组合，Mume 运行必须基于一个特定的配置组。配置组的设定主要是为了便于用户在不同网络环境或者基于不同目的（调试或者正常使用等）能够方便的切换。配置组修改后，需要重启 VPN 才能生效。

导入配置

Mume 支持导入预设的配置（代理、规则、配置组等），您可以在『更多（ More）』面板导入配置。

目前支持的导入方式：

  • 提供一个可访问的 URL 地址下载配置或是代理的文本表达形式；
  • 扫描二维码（二维码可以是一个 SS 或 SSR 的配置，或者 URL 地址）
